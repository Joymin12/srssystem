# Implementation Log

## UC-01 로그인

- 사용자 ID와 비밀번호 입력 흐름을 구성했다.
- 로그인 요청을 서버 API로 전달하고 결과를 확인한다.
- 로그인 성공 시 사용자 ID, 이름, 역할 정보를 세션에 저장한다.
- 역할에 따라 학생, 교수, 조교가 사용할 수 있는 화면을 분기한다.
- 로그인 실패 시 사용자에게 실패 메시지를 안내한다.

## UC-02 강의실 및 강의 현황 조회

- 강의실 목록을 조회해 사용자가 조회 대상을 선택할 수 있게 구성했다.
- 기준 날짜와 일간, 주간, 월간 조회 옵션을 입력받는다.
- 클라이언트에서 `/rooms/{roomId}/status` API를 호출한다.
- 서버에서 강의실 존재 여부와 건물 ID 일치 여부를 확인한다.
- 강의 시간표와 승인된 예약 현황을 함께 조회해 응답한다.
- 클라이언트는 `LECTURE`, `RESERVATION` 응답 라인을 분리해 각각의 표에 표시한다.

## UC-02 조회 조건 보완

- 날짜 파라미터가 비어 있을 때 서버에서 현재 날짜를 기준으로 처리하도록 정리했다.
- 조회 범위가 daily인 경우 기준 날짜의 요일과 일치하는 강의만 표시한다.
- weekly 조회는 기준 날짜 전후 범위에 포함되는 예약만 표시한다.
- monthly 조회는 기준 날짜와 같은 연도, 같은 월의 예약만 표시한다.
- 승인된 예약만 현황에 포함되도록 상태 필터를 유지한다.
