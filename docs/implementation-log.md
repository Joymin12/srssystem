# Implementation Log

## UC-01 로그인

- 사용자 ID와 비밀번호 입력 흐름을 구성했다.
- 로그인 요청을 서버 API로 전달하고 결과를 확인한다.
- 로그인 성공 시 사용자 ID, 이름, 역할 정보를 세션에 저장한다.
- 역할에 따라 학생, 교수, 조교가 사용할 수 있는 화면을 분기한다.
- 로그인 실패 시 사용자에게 실패 메시지를 안내한다.

## UC-02 강의실 및 강의 현황 조회

- 강의실 목록을 조회해 사용자가 조회 대상을 선택할 수 있게 구성했다.
- 기준 날짜와 일간, 주간, 월간 조회 옵션을 입력받는다.
- 클라이언트에서 `/rooms/{roomId}/status` API를 호출한다.
- 서버에서 강의실 존재 여부와 건물 ID 일치 여부를 확인한다.
- 강의 시간표와 승인된 예약 현황을 함께 조회해 응답한다.
- 클라이언트는 `LECTURE`, `RESERVATION` 응답 라인을 분리해 각각의 표에 표시한다.

## UC-02 조회 조건 보완

- 날짜 파라미터가 비어 있을 때 서버에서 현재 날짜를 기준으로 처리하도록 정리했다.
- 조회 범위가 daily인 경우 기준 날짜의 요일과 일치하는 강의만 표시한다.
- weekly 조회는 기준 날짜 전후 범위에 포함되는 예약만 표시한다.
- monthly 조회는 기준 날짜와 같은 연도, 같은 월의 예약만 표시한다.
- 승인된 예약만 현황에 포함되도록 상태 필터를 유지한다.

## UC-03 빈 강의실 조회

- 건물, 날짜, 요일, 시작 교시, 종료 교시를 조회 조건으로 받는다.
- 클라이언트에서 빈 강의실 조회 API를 호출한다.
- 서버는 조회 조건에 해당하는 건물의 강의실 목록을 가져온다.
- 강의 시간표와 충돌하는 강의실을 제외한다.
- 승인된 예약과 충돌하는 강의실을 제외한다.
- 남은 강의실을 예약 가능한 강의실 목록으로 반환한다.

## UC-03 입력값 검증 보완

- 건물이 선택되지 않은 경우 검색을 진행하지 않도록 처리했다.
- 날짜 형식이 올바르지 않으면 사용자에게 안내 메시지를 표시한다.
- 시작 교시와 종료 교시는 숫자로만 입력되도록 검증한다.
- 종료 교시가 시작 교시보다 빠른 경우 조회를 차단한다.
- 날짜를 기준으로 요일을 자동 계산해 조회 조건 불일치를 줄였다.
- 서버에서도 필수 조회 조건이 누락되면 오류 응답을 반환한다.

## UC-04 강의실 예약 신청

- 학생과 교수가 강의실 예약을 신청할 수 있는 화면을 구성했다.
- 예약 신청 API를 통해 신청자, 역할, 강의실, 날짜, 교시 정보를 전달한다.
- 예약 목적, 사용 인원, 동반자 정보를 입력받는다.
- 예약 요청은 먼저 유효성 검사를 거친다.
- 강의 시간표와 충돌하는 예약 신청을 제한한다.
- 조건을 통과한 예약은 `PENDING` 상태로 저장한다.

## UC-05 예약 목록 및 상세 조회

- 사용자 ID 기준으로 본인의 예약 목록을 조회한다.
- 예약 목록에는 신청자, 강의실, 날짜, 교시, 목적, 상태를 표시한다.
- 예약 ID를 기준으로 상세 정보를 조회할 수 있게 구성했다.
- 학생과 교수는 자신의 예약 내역을 확인할 수 있다.
- 예약 상태 변경 결과가 목록 화면에 반영되도록 서버 응답을 연결했다.

## UC-06 예약 취소

- 사용자가 본인의 예약을 선택해 취소할 수 있게 구성했다.
- 예약 취소 요청에는 사용자 ID, 예약 ID, 취소 사유를 전달한다.
- 서버는 예약 소유자와 요청자가 일치하는지 확인한다.
- 이미 취소된 예약은 다시 취소하지 못하도록 제한한다.
- 취소가 완료되면 예약 상태를 `CANCELED`로 변경한다.
- 취소 결과는 알림 생성 로직과 연결된다.

## UC-07 예약 승인 및 거절

- 조교 사용자가 대기 예약 목록을 조회할 수 있게 구성했다.
- 조교 권한을 확인한 뒤 승인 또는 거절 요청을 처리한다.
- 승인된 예약은 `APPROVED` 상태로 변경한다.
- 거절된 예약은 `REJECTED` 상태로 변경하고 사유를 저장한다.
- 교수 예약 승인 시 충돌하는 학생 예약을 취소 처리한다.
- 승인과 거절 결과는 사용자 알림으로 전달된다.

## UC-08 예약 결과 알림 확인

- 사용자 ID 기준으로 알림 목록을 조회한다.
- 읽지 않은 알림은 로그인 후 팝업으로 표시한다.
- 새 알림이 중복으로 표시되지 않도록 표시된 알림 ID를 관리한다.
- 알림 목록 화면에서 알림 내용을 확인할 수 있다.
- 알림 읽음 처리 API를 통해 확인 상태를 저장한다.

## UC-09 강의실 및 강의 시간 관리

- 조교 사용자가 강의실 목록을 관리할 수 있게 구성했다.
- 강의실 등록 API를 통해 건물, 강의실 번호, 수용 인원을 저장한다.
- 강의실 삭제 시 관련 강의 시간표도 함께 정리한다.
- 강의 시간표 등록 API를 통해 요일과 교시 정보를 저장한다.
- 강의 시간표 삭제 API를 통해 잘못 등록된 강의 정보를 제거한다.
- 관리 화면과 서버 관리 API를 연결했다.
